Chapter 2: Derive Macros

2.1 Fundamentals of Derive Macros

Derive macros are the most common type of procedural macro. They allow users to automatically implement traits for their types by adding #[derive(TraitName)]. Under the hood, the derive macro parses the annotated type (struct/enum/union), generates the trait implementation code, and injects it into the compilation process.

To define a derive macro, use the #[proc_macro_derive(TraitName)] attribute on a function. The function takes a TokenStream representing the annotated type, parses it with syn, generates the trait implementation with quote, and returns the generated TokenStream.

2.2 Implementing a Custom Derive Macro

Let’s create a derive macro MyDebug that generates a simple debug implementation (similar to the built-in Debug trait). First, add the trait definition (in a separate crate or the same binary crate) and the derive macro in the proc-macro crate.

Step 1: Define the MyDebug trait in a shared crate (e.g., my-traits):

#![allow(unused)]
fn main() {
// my-traits/src/lib.rs
pub trait MyDebug {
    fn my_debug(&self) -> String;
}

}

Step 2: Implement the derive macro in my-proc-macros/src/lib.rs:

#![allow(unused)]
fn main() {
// src/lib.rs
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Fields, Ident};

// Derive macro for MyDebug trait
#[proc_macro_derive(MyDebug)]
pub fn derive_my_debug(input: TokenStream) -> TokenStream {
    // Parse the input into a DeriveInput (represents a struct/enum/union)
    let input = parse_macro_input!(input as DeriveInput);
    
    // Extract the name of the type (e.g., "User" for struct User { ... })
    let type_name = input.ident;
    
    // Generate the trait implementation based on the type's fields
    let impl_block = match input.data {
        syn::Data::Struct(data) => match data.fields {
            Fields::Named(fields) => {
                // For structs with named fields (e.g., struct S { a: i32, b: String })
                let field_names: Vec<Ident> = fields.named.iter()
                    .map(|field| field.ident.as_ref().unwrap())
                    .collect();
                
                quote! {
                    impl MyDebug for #type_name {
                        fn my_debug(&self) -> String {
                            format!(
                                "{} {{ {} }}",
                                stringify!(#type_name),
                                #(
                                    format!("{}: {:?}", stringify!(#field_names), &self.#field_names)
                                ),*
                            )
                        }
                    }
                }
            }
            Fields::Unnamed(fields) => {
                // For structs with unnamed fields (e.g., struct S(i32, String))
                let field_indices: Vec<usize> = (0..fields.unnamed.len()).collect();
                
                quote! {
                    impl MyDebug for #type_name {
                        fn my_debug(&self) -> String {
                            format!(
                                "{}({})",
                                stringify!(#type_name),
                                #(
                                    format!("{:?}", &self.#field_indices)
                                ),*
                            )
                        }
                    }
                }
            }
            Fields::Unit => {
                // For unit structs (e.g., struct S;)
                quote! {
                    impl MyDebug for #type_name {
                        fn my_debug(&self) -> String {
                            stringify!(#type_name).to_string()
                        }
                    }
                }
            }
        },
        syn::Data::Enum(_) | syn::Data::Union(_) => {
            // For simplicity, we'll only support structs in this example
            panic!("MyDebug is only supported for structs");
        }
    };
    
    TokenStream::from(impl_block)
}

}

Step 3: Use the derive macro in a binary crate:

// src/main.rs
use my_proc_macros::MyDebug;
use my_traits::MyDebug as MyDebugTrait;

#[derive(MyDebug)]
struct User {
    name: String,
    age: u32,
}

#[derive(MyDebug)]
struct Point(i32, i32);

#[derive(MyDebug)]
struct Empty;

fn main() {
    let user = User { name: "Alice".to_string(), age: 30 };
    println!("{}", user.my_debug()); // Prints "User { name: "Alice", age: 30 }"
    
    let point = Point(10, 20);
    println!("{}", point.my_debug()); // Prints "Point(10, 20)"
    
    let empty = Empty;
    println!("{}", empty.my_debug()); // Prints "Empty"
}

2.3 Derive Macros with Attributes

You can extend derive macros to accept custom attributes (e.g., #[my_debug(skip)] on fields) to customize code generation. This requires using #[proc_macro_derive(TraitName, attributes(attr_name))] to declare the allowed attributes, then parsing those attributes with syn.

Example: Modify MyDebug to skip fields annotated with #[my_debug(skip)]:

#![allow(unused)]
fn main() {
// my-proc-macros/src/lib.rs
use syn::{Meta, NestedMeta, Path};

// Update the derive macro attribute to allow `my_debug` field attributes
#[proc_macro_derive(MyDebug, attributes(my_debug))]
pub fn derive_my_debug(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let type_name = input.ident;
    
    let impl_block = match input.data {
        syn::Data::Struct(data) => match data.fields {
            Fields::Named(fields) => {
                let filtered_fields: Vec<&syn::Field> = fields.named.iter()
                    .filter(|field| !is_skip_field(field))
                    .collect();
                
                let field_names: Vec<Ident> = filtered_fields.iter()
                    .map(|field| field.ident.as_ref().unwrap())
                    .collect();
                
                quote! {
                    impl MyDebug for #type_name {
                        fn my_debug(&self) -> String {
                            format!(
                                "{} {{ {} }}",
                                stringify!(#type_name),
                                #(
                                    format!("{}: {:?}", stringify!(#field_names), &self.#field_names)
                                ),*
                            )
                        }
                    }
                }
            }
            // ... (keep Unnamed and Unit cases unchanged)
        },
        // ... (keep Enum/Union panic)
    };
    
    TokenStream::from(impl_block)
}

// Helper function to check if a field has #[my_debug(skip)]
fn is_skip_field(field: &syn::Field) -> bool {
    for attr in &field.attrs {
        // Check if the attribute is `#[my_debug(...)]`
        if attr.path().is_ident("my_debug") {
            // Parse the attribute's arguments
            if let Ok(Meta::List(meta_list)) = attr.parse_meta() {
                for nested_meta in meta_list.nested {
                    if let NestedMeta::Meta(Meta::Path(path)) = nested_meta {
                        if path.is_ident("skip") {
                            return true;
                        }
                    }
                }
            }
        }
    }
    false
}

}

Use the skip attribute in the binary crate:

#[derive(MyDebug)]
struct User {
    name: String,
    #[my_debug(skip)]
    password: String, // Skipped in my_debug output
    age: u32,
}

fn main() {
    let user = User {
        name: "Alice".to_string(),
        password: "secret".to_string(),
        age: 30,
    };
    println!("{}", user.my_debug()); // Prints "User { name: "Alice", age: 30 }"
}

2.4 Practice Exercises

  1. Extend the MyDebug derive macro to support enums. For enums, generate a my_debug implementation that prints the variant name and its fields (if any).

  2. Add a #[my_debug(rename = "new_name")] attribute to allow renaming fields in the my_debug output (e.g., #[my_debug(rename = "full_name")] name: String prints “full_name: …”).

  3. Create a derive macro DefaultValue that implements a trait DefaultValue with a method default_value() -> Self returning a default instance of the type. For structs, use default values (0 for integers, empty strings for String, etc.).
